# 07 · Przykładowe jadłospisy

> Jadłospisy z dokumentów źródłowych (głównie z prezentacji M. Jędruszczak-Kępka oraz ulotki Lublin 2019). Stanowią **„złoty standard"** — dane do walidacji rekomendacji modelu.

## Wariant 1: Dieta lekkostrawna, wysokobiałkowa, **płynna** (zmiksowana)

Wszystko miksowane do 400 ml.

| Posiłek | Skład |
|---------|-------|
| **Śniadanie** | Kasza manna na mleku + serek homogenizowany + żółtko + masło ekstra · 400 ml mix |
| **II śniadanie** | Kleik ryżowy + ser twarogowy + sok z marchwi i jabłka + miód · 400 ml mix |
| **Obiad** | Zupa jarzynowa z mięsem + cukinia gotowana + oliwa z oliwek · 400 ml mix |
| **Podwieczorek** | Budyń z mleka + jajko + banan + suchar · 400 ml mix |
| **Kolacja** | Ryba gotowana + ryż + marchew gotowana + masło ekstra · 400 ml mix |

**Wskazania:** dysfagia ciężka, zapalenie jamy ustnej / przełyku ciężkie, stan po operacji w obrębie głowy/szyi/przełyku.

## Wariant tygodniowy z ulotki Lublin 2019 (opcja standard)

## Wnioski strukturalne (do feature engineeringu)

Z analizy jadłospisów wynika, że typowy dzień zawiera:

1. **Śniadanie** — bazujące na **kaszy / płatkach na mleku** + **białko** (jajko / pasta mięsna / wędlina drobiowa) + pieczywo z masłem + **ciepły napój ziołowy**.
2. **II śniadanie** — **owoc + nabiał** (jogurt / kefir / serek / budyń) lub coś słodkiego (mus / jabłko pieczone).
3. **Obiad** — **zupa** + **białko zwierzęce** (gotowane/pieczone w folii) + **dodatek węglowodanowy** (ziemniaki, ryż, makaron) + **warzywo gotowane** + kompot.
4. **Podwieczorek** — kanapka z chudą wędliną / pasta + napój / owoc.
5. **Kolacja** — **lekkie białko** (twarożek, ryba, jajka, tymbalik, jajecznica na parze) + pieczywo + warzywo + lekka herbata.

### Cechy posiłku do wyciągnięcia z jadłospisu (dla modelu)

```yaml
meal_features:
  - meal_slot: enum [sniadanie, II_sniadanie, obiad, podwieczorek, kolacja]
  - components:
      protein_source: enum [drob, indyk, ryba, jajka, twarog, jogurt, kefir, miesa_chude_wieprz, cielecina, brak]
      carb_source: enum [ryz, makaron, ziemniaki, kasza_drobna, kasza_gruba, pieczywo_pszenne, pieczywo_pelnoziarniste, brak]
      vegetable: list [marchew, dynia, szpinak, cukinia, brokul_gotowany, burak_gotowany, ...]
      fat_source: enum [maslo, oliwa, olej_rzepakowy, olej_lniany, awokado, brak]
      dairy: enum [mleko, jogurt, kefir, twarog, serek_homo, brak]
      fruit: list [banan, jablko_pieczone, mus_jablkowy, suszone_sliwki, brzoskwinia, ...]
  - liquid: enum [herbata_malinowa, herbata_zielona, herbata_mietowa, kompot, mleko, kefir, kakao_na_wodzie, woda]
  - consistency: enum [standardowa, papkowata, plynna, krem]
  - estimated_kcal: int
  - estimated_protein_g: int
  - tags_symptom_compatible: list[SymptomKey]
```

To jest gotowy schemat dla `frontend/src/types.ts → interface Meal`.
